# Security Policy

Please report security-sensitive issues privately when disclosure would put users
or devices at risk.

Do not attach private keys, retail keys, Amiibo dumps, or personal device data to
public issues. If a report needs logs, remove secrets and personal identifiers
first.

Security-sensitive areas include:

- DFU signing and release artifacts,
- bootloader behavior,
- BLE update flows,
- NFC bounds and parser hardening,
- settings migration and persistence,
- GitHub Actions secrets.
